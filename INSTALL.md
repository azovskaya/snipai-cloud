# SnipAI v2.0 — Установка за 5 шагов

## Требования
- macOS / Linux / Windows (с WSL2)
- 4 GB свободного места на диске
- Интернет для первой загрузки моделей

---

## ШАГ 1: Установить Docker Desktop

### macOS:
1. Откройте браузер → [https://www.docker.com/products/docker-desktop/](https://www.docker.com/products/docker-desktop/)
2. Нажмите **"Download for Mac"** (выберите Apple Silicon или Intel)
3. Откройте скачанный `.dmg` файл → перетащите Docker в Applications
4. Запустите Docker из Applications
5. Дождитесь появления иконки кита в строке меню ✅

**Проверка:**
```bash
docker --version
# Должно показать: Docker version 24.x или выше

docker-compose --version
# Должно показать: Docker Compose version v2.x
```

---

## ШАГ 2: Настроить проект

```bash
# Перейти в папку проекта
cd /путь/к/вашей/папке/SnipAI

# Создать .env файл из шаблона
cp .env.example .env

# Открыть .env и вставить свой Google API ключ
nano .env
# или: open -e .env  (откроет в TextEdit)
```

**В файле .env замените:**
```
GOOGLE_API_KEY=your_google_api_key_here
```
**На ваш реальный ключ** (получить: https://aistudio.google.com/app/apikey)

---

## ШАГ 3: Структура папки проекта

Убедитесь что структура выглядит так:
```
SnipAI/
├── Dockerfile
├── docker-compose.yml
├── Makefile
├── requirements.txt
├── .env                ← заполненный вами
├── .env.example
├── src/
│   ├── core.py
│   ├── ingest.py
│   └── query.py
└── data/
    └── snip_zhilye.docx   ← ваши документы сюда
```

---

## ШАГ 4: Первый запуск

```bash
# 4.1 Собрать Docker образ (один раз, ~5-10 минут)
# Скачивает Python 3.11, устанавливает все зависимости
make build

# 4.2 Запустить Qdrant (векторная БД)
make start

# 4.3 Проиндексировать документы из ./data/
# Первый раз скачает модель эмбеддингов (~117MB), ~2 минуты
make ingest
```

**Что происходит при `make ingest`:**
1. Загружается модель `multilingual-e5-small` (только первый раз)
2. Читаются все PDF/DOCX/DOC из папки `data/`
3. Извлекается текст + таблицы
4. Создаются векторные эмбеддинги
5. Сохраняются в Qdrant

## Ежедневное использование

```bash
# Добавить новые документы в data/ и переиндексировать
# (уже проиндексированные файлы будут пропущены)
make ingest

# Задать вопрос
make query

# Остановить контейнеры когда не нужны
make stop

# Снова запустить
make start && make query
```

---

## Если что-то пошло не так

```bash
# Посмотреть логи
make logs

# Перезапустить
make stop && make start

# Пересобрать образ (если изменили код)
make build
```

## Перенос на другой компьютер

```bash
# На новом компьютере:
# 1. Установить Docker Desktop (Шаг 1)
# 2. Скопировать папку SnipAI полностью
# 3. Создать .env с ключами
# 4. make build && make start && make ingest
```

Данные Qdrant хранятся в Docker Volume — они НЕ копируются.
Поэтому нужна переиндексация на новой машине.
Файлы из `data/` — ваши документы, копируются вместе с проектом.
